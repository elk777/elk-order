## 已修复

1. 订单与请求高严重缺陷
   - `stores/order.js`：订单详情缓存归一化幂等；日期筛选改用 `rawOrderTime` 标准日期；`hasMore` 统一进 store；并发请求只允许最后一次落库；loadMore 失败回退页码；订单持久化只保留筛选状态。
   - `pages/order/component/DateFiltering.vue`、`OrderList.vue`：日期/状态/刷新/加载更多均重新请求后端并使用 store 状态。
   - `pages/cart/AffirmOrder.vue`：下单成功后清购物车失败不再误报提交失败；延迟跳转定时器在卸载时清理。
   - `utils/request.js`：保留显式 `data: null`；token 刷新失败统一唤醒等待队列；开发日志对 token、phone、openid 等字段脱敏。

2. 上传与菜谱编辑稳定性
   - `components/Upload/index.vue`：兼容 uView 删除事件对象；同步 `update:fileList` / `change`；移除调试残留。
   - `pages/recipe/redact.vue`：封面上传加并发锁；上传中禁止提交；禁止提交本地临时路径；上传失败只移除本次临时文件；延迟返回定时器清理；清理调试日志。
   - `pages/recipe/details.vue`：食材/步骤使用稳定 key；失败延迟返回可清理；复制清单补空格分隔。
   - `pages/recipe/hook/useListOperations.js`：移除调试日志。

3. 组件数据流与登录/购物车边界
   - `components/CateList/index.vue`：数量变更改为事件上行，避免直接修改 computed 派生对象。
   - `pages/cart/component/CateTab.vue`：加入购物车保留真实菜谱封面；菜单加载保持未登录短路；共享菜谱只对可管理项显示编辑/删除。
   - `stores/recipe.js`：购物车数量按字符串 ID 比较；持久化不再保存购物车列表。
   - `stores/user.js`、`utils/auth.js`：持久化剔除 `profile.phone`；显式退出/401 时清本地登录缓存，初始化归一化不误删。

4. 性能、安全和工程清理
   - `pages/home/index.vue`：壁纸初始化与资料刷新并行，仍保证资料刷新早于邀请码消费。
   - `components/Search/index.vue`、`pages/sort/index.vue`：搜索输入 400ms 防抖；菜单 onShow 刷新补异常捕获。
   - `pages/order/component/OrderButton.vue`：再来一单延迟跳转在卸载时清理。
   - `main.js`、`App.vue`：增加 Vue/uni 全局错误日志兜底；移除 App hide 调试日志。
   - `manifest.json`：恢复微信小程序 `urlCheck: true`；收敛 App 端高风险权限。
   - 删除未引用死文件：`apis/order/index.js`、`pages/home/component/LegacyHomeContent.vue`。
