# 前端代码审查报告（hkt-applet · 偏爱厨房小程序）

> 审查日期：2026-06-25
> 技术栈：uni-app + Vue3（`<script setup>`）+ Pinia + uview-plus，纯 JavaScript
> 审查范围：`pages/ apis/ api/ components/ stores/ utils/ hooks/ config/ common/` 及 `App.vue / main.js / static/`，约 18,000 行源码
> 方式：6 个独立子代理分别审查【安全 / 代码质量 / 缺陷 / 性能 / 测试稳定性 / 可维护性】，主代理去重合并、按优先级整理

---

## 一、总体评估

| 维度 | 评级 | 一句话结论 |
|---|---|---|
| 安全 | 良好（B+） | 无硬编码密钥、无 XSS/eval、上传走后端签名直传；主要问题是 token 明文存储与「越权防护只在前端」 |
| 代码质量 | 中上（B-） | `<script setup>` 100% 统一、request 封装成熟；扣分在一致性纪律（双目录、硬编码主题色、重复封装） |
| 缺陷/漏洞 | 中-偏高 | 存在 8 个高严重必现/易现 bug，集中在数据归一化、异步状态复位、第三方组件契约 |
| 性能 | 中上（B） | 资源精简、分页/定时器清理到位；主要风险是持久化全量写盘、串行请求瀑布、全局缺防抖节流 |
| 测试稳定性 | 不及格（2/10） | 0 测试、0 CI、0 Lint、0 全局错误兜底，质量完全依赖人工真机回归 |
| 可维护性 | 中上（6.5/10） | 底子好（文档/配置/低耦合），欠债在「改了一半」的目录迁移和缺失工具链 |

**一句话总览**：基础架构（`utils/request.js` 封装、Pinia 分层、config 集中、README 文档）明显优于同类小程序，**但存在三类系统性欠债**：① 半途而废的重构（`api/`+`apis/` 双目录）；② 工程化安全网缺失（无测试/Lint/类型/CI/全局错误兜底）；③ 一批必现功能 bug 未被发现（恰恰因为没有测试）。

## 二、跨维度共性根因（多个代理都指向的问题，优先处理）

这些问题被 2 个以上维度同时点名，修一处可多维度受益：

1. **`api/`（单数）与 `apis/`（复数）双目录并存** — 被「代码质量 / 可维护性」同时列为最严重。`apis/order/index.js` 是空壳、`apis/cart/` 是空目录，真正接口在 `api/`。新人无法判断接口该写哪、IDE 自动导入易选错。是 `optimize-request-wrapper` 任务只迁移了 3 个模块的半成品。
2. **零测试 / 零 Lint / 零 CI / 零全局错误兜底** — 「测试稳定性」判为不及格，同时是「可维护性 / 缺陷」的根因放大器：正因为没有测试，下面第三节那批必现 bug 才会进生产。
3. **超大文件** — `pages/home/index.vue` **1101 行**、`UserEdit.vue` 846 行、`CateTab.vue` 710 行、`redact.vue` 692 行。被「质量 / 可维护性 / 性能」共同点名。
4. **主题色 `#FF5C8D` 硬编码 71 处** — 已有 `COLOURS['theme-color']` 常量却未复用；换肤需全局搜替。
5. **`console.*` 残留 ~77 处**（`redact.vue` 占 12-16）— 被「质量 / 性能 / 安全 / 可维护性」共同点名，业务层无生产环境守卫。
6. **持久化策略粗放** — Pinia persist 未限定字段，分页列表全量同步写盘（性能）；token+手机号明文落盘（安全）。

## 四、安全问题

**整体良好**：无硬编码又拍云/AppSecret/AK-SK 密钥，上传走后端签名直传（前端不持密钥），隐私配置已 gitignore 且未入库，无 `eval`/`v-html`/明文证书跳过，传输默认 HTTPS。**未发现高危**。需改进项：

| 严重度 | 问题 | 位置 | 修复 |
|---|---|---|---|
| 中 | token + 手机号 + uuid **明文持久化**到本地存储 | `stores/user.js:104-112` | token 单独短有效期存储 + refresh 机制；持久化内容轻量加密；登出显式 `removeStorageSync('user')` |
| 中 | **越权防护完全依赖前端路由守卫**，信任前端传入的 `userId/uuid/inviteUid` | `utils/auth.js`、`utils/invite.js:97` | 所有敏感写接口的鉴权与归属校验**必须后端基于 token 完成**；前端守卫仅作体验 |
| 中 | 微信请求域名校验被关闭 `urlCheck:false` | `manifest.json:55-60` | 生产恢复 `urlCheck:true`，公众平台配置合法域名 |
| 低 | 非生产环境完整打印请求/响应（含登录 token） | `utils/request.js:498-504` | 对 `Authorization/token/phone` 脱敏 |
| 低 | 残留调试日志输出文件对象、内网/穿透地址注释、请求无防重放签名、App 端权限过宽 | `Upload/index.vue`、`config/index.js:9`、`manifest.json:26-42` | 清理日志与注释、收敛权限、敏感写操作加 HMAC+nonce |

**优先级**：先核实后端是否对敏感写接口独立鉴权（越权，影响最大）→ token 存储加固 → 恢复 urlCheck + 日志脱敏。

## 六、测试稳定性

**现状（量化）**：单元测试 **0**、E2E **0**、测试框架 **0**、ESLint/Prettier **0**、TypeScript **0**、CI **0**、全局错误处理（`onError`/`errorHandler`/`onUnhandledRejection`）**0 处**。`package.json` 的 `scripts`/`devDependencies` 均为空。

**关键风险**：含支付/下单/上传/鉴权敏感路径却零自动化测试；无全局崩溃边界，未捕获 Promise 异常静默丢失、无上报无提示；`refreshToken` 重放只做一次、上传无重试无 token 重放；核心逻辑与 `uni.*` 强耦合导致可测性差；`config/privacy.js` 靠手动注释切换 URL，易把生产/内网地址打错包。

**建议测试栈**：Vitest + `@vue/test-utils` + `@pinia/testing`，`vitest.setup.js` 全局 mock `uni`；配 ESLint(`eslint-plugin-vue`)+Prettier + 最小 GitHub Actions CI（install→lint→vitest run）。E2E 用微信开发者工具 minitest，优先级后置。

**最该优先补测的 5 个模块**：①`utils/request.js`（ROI 最高：状态码归一/401 刷新重放/重试/去重）②`utils/auth.js`（路由鉴权纯逻辑）③ 下单购物车（`getOrderActions` 动作矩阵+状态流转）④ 上传链路（扩展名/MIME 校验+签名失败兜底）⑤`stores/user.js`（`normalizeProfile` 字段兜底）。

**并行加固**：`main.js`/`App.vue` 增加 `app.config.errorHandler` + `uni.onError` + `onUnhandledRejection`；用构建变量驱动 `baseUrl` 替代手动注释切换。

## 附录·缺陷全量（中/低严重）

中严重：M1 loadMore 失败页码不回退 → 跳页丢数据；M2 OrderList `hasMore` 外部刷新不复位；M3 订单页首屏并发重复拉取竞态；M4 编辑模式空清单被静默忽略；M5 菜谱图片上传无并发锁+`pop()` 错位；M6 封面上传未完成可提交本地路径；M7 `parseResponse` 用 `data.data ?? data`，`data:null` 时吞整个信封（`utils/request.js:404`）；M8 首页 onShow 并发竞态+uuid 未就绪落兜底 key；M9 登录跳转 setTimeout 未清理；M10 首页引导定时器仅 onUnmounted 清理（tab 切走应 onHide）；M11 跨午夜"今日"错位；M12 IntegralDetail 仅取第一页+切 tab 竞态；M13 sort 页 onShow 未 await/未 catch；M14 又拍云上传未校验签名 code+进度回调被忽略+大视频超体积；M15 Tabbar 切换无越界校验；M16 Upload `JSON.stringify` 比对误判覆盖；M17 编辑菜谱被强制先建分类。

低严重：L1 `hasMore` 整页边界多发空请求；L2 `getCartQuantity` 用 `===` 比 id；L3 多处跳转 setTimeout 未清理；L4 CartPopup 清空未 await/catch；L5 金额计算 NaN 隐患（建议以分为单位）；L6 `genderText/roleText` 用 `=== 0/1` 与字符串不匹配；L7 首页 userType 三态压成两态；L8 `amountDetail` 未知 type 返回 undefined；L9 `formatTime/formatDate` 非法日期无防护；L10 日历可无限翻未来月；L11 业务判断依赖中文文案 `includes("已签到")`；L12 dayjs 解析非 ISO 在 iOS 可能 Invalid；L13 media 协议相对 URL 永不升级 https；L14 `copyIngredients` 与预览分隔不一致；L15 多处可选链/防御缺失；L16 订阅消息并发竞态；L17 残留 console.log。
